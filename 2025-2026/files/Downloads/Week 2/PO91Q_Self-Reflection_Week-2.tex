\section{Answers}\label{answers}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Why is it good practice to always use an RScript, rather than simply
  entering your commands into the console in R?

  \begin{itemize}
  \tightlist
  \item
    Ensures reproducibility

    \begin{itemize}
    \tightlist
    \item
      Scripts provide a saved record of all your steps, making it easy
      to reproduce or rerun your analysis.
    \end{itemize}
  \item
    Improves organization

    \begin{itemize}
    \tightlist
    \item
      You can structure your code with comments, sections, and logical
      flow, making complex work manageable.
    \end{itemize}
  \item
    Easier debugging

    \begin{itemize}
    \tightlist
    \item
      You can edit and re-run parts of your code without retyping
      everything or losing previous work.
    \end{itemize}
  \item
    Saves time

    \begin{itemize}
    \tightlist
    \item
      Once your script is written, you can re-run the entire analysis
      with one click or command---ideal for iterative work.
    \end{itemize}
  \item
    Better collaboration and communication

    \begin{itemize}
    \tightlist
    \item
      Sharing an R script allows others to understand, verify, or extend
      your work.
    \end{itemize}
  \end{itemize}
\item
  Give an example of an unordered factor variable and an ordered one.

  \begin{itemize}
  \tightlist
  \item
    Unordered factor variable (nominal variable)

    \begin{itemize}
    \tightlist
    \item
      ``fruit'' with categories ``apple'', ``banana'', ``orange''
    \end{itemize}
  \item
    Ordered factor variable (ordinal variable)

    \begin{itemize}
    \tightlist
    \item
      ``education\_level'' with categories ``High School'',
      ``Bachelor'', ``Master'', ``PhD'' \newpage
    \end{itemize}
  \end{itemize}
\item
  Why are descriptive statistics useful?

  \begin{itemize}
  \tightlist
  \item
    Summarize data quickly

    \begin{itemize}
    \tightlist
    \item
      They condense large datasets into a few key numbers (e.g., mean,
      median, standard deviation) for quick interpretation.
    \end{itemize}
  \item
    Understand data distribution

    \begin{itemize}
    \tightlist
    \item
      Measures like range, variance, skewness, and kurtosis reveal the
      shape and spread of your data.
    \end{itemize}
  \item
    Identify patterns and trends

    \begin{itemize}
    \tightlist
    \item
      Descriptive statistics highlight central tendencies and
      variability, helping detect patterns or irregularities.
    \end{itemize}
  \item
    Spot errors or outliers

    \begin{itemize}
    \tightlist
    \item
      Unusual values in summary statistics can signal data entry
      mistakes or meaningful anomalies.
    \end{itemize}
  \item
    Prepare for further analysis

    \begin{itemize}
    \tightlist
    \item
      Descriptive stats are a critical first step before performing
      inferential statistics or building models.
    \end{itemize}
  \end{itemize}
\item
  Give an example of a situation where you would need to transform a
  character variable into a factor variable.

  \begin{itemize}
  \tightlist
  \item
    Whenever you need the values of a variable as levels:

    \begin{itemize}
    \tightlist
    \item
      gender: ``male'', ``female''
    \item
      month: ``January'', ``February'', etc.
    \item
      education: ``High School'', ``Bachelor'', ``Master'', ``Doctoral''
    \item
      region: ``north'', ``south'', etc.
    \end{itemize}
  \end{itemize}
\item
  Why should you always create a new variable, instead of overwriting an
  existing one?

  \begin{itemize}
  \tightlist
  \item
    Preserve original data: Keeping the original variable intact ensures
    you can always refer back to the unmodified data if needed.
  \item
    Avoid accidental data loss: Overwriting removes the original
    content, which can't be recovered unless previously saved or backed
    up.
  \item
    Improve reproducibility: Creating new variables helps others (or
    future you) understand each step of the transformation process.
  \item
    Enhance debugging: If something goes wrong, it's easier to trace the
    issue when intermediate variables are preserved.
  \item
    Facilitate comparison: You can compare the original and modified
    data side-by-side, which helps with validation and analysis.
  \item
    Encourage cleaner code: Using new variables reflects clearer logic
    and makes your code more readable and maintainable.
  \end{itemize}
\end{enumerate}
