Methods, Methods, Methods

In the coming weeks, you will find this section on the Companion to help you apply the material we cover in the lecture in R. It will contain an RScript for some preliminary data preparation. This is not an actual part of introducing the method, but you are certainly encouraged to read it and to try and understand it.

These sections will work with the American National Election Studies (ANES), and to be more precise with the pilot survey conducted before the 2020 presidential election. You can download the required data set by following this link. The actual data set itself is available here.

Data Prep

Place the ANES data in a folder which you will be using as a working directory for this session. Open the “Code for Data Preparation” below, and copy this into an RScript. Remember to adjust the working directory with the setwd() command at the beginning. Then run the RScript and you will be ready to proceed to the video.

Code for Data Preparation 
######################################
# MMM - Week 1 - Data Preparation
######################################


# Set WD
setwd()

# Load packages

library(tidyverse)

# Load data set

anes <- read_csv("anes.csv")

# Get rid of missing values for variables used in analysis today

## 999 is equivalent to NA, so needs to be recoded
anes$fttrump1 <- with(anes, replace(fttrump1, fttrump1 == 999, NA)) 

anes <- filter(anes, 
               !is.na(fttrump1),
               !is.na(sex))

# turn support for Trump `fttrump1` into ordered factor with three levels

anes <- anes %>% 
  mutate(trump=
           ordered(
             cut(fttrump1, breaks=c(0, 33, 66, 100), 
                 labels=c("low","medium","high"))))

# Label variable `sex`
anes$sex <- factor(anes$sex)

anes <- anes %>% 
  mutate(sex=
           recode(sex,"1"="Male",
                  "2"="Female"))

# save data set for use in video

write.csv(anes, "anes_week1.csv")

You can copy the code from this page by hovering over the code chunk and clicking the icon in the top-right hand corner. You can then paste it into your RScript.

Conceptual Exercises

Crosstabs

I have given you an example of a cross-tabulation in the lecture. Consider the following Table:

 

Mode of Transport
Year of Study Bike Bus Car Total
PAIS 10 5 5 20
CIM 8 12 10 30
Total 18 17 15 50

 

Calculate the Expected Values and fill in the following table:

 

Mode of Transport
Year of Study Bike Bus Car Total
PAIS
CIM
Total

 

  1. Calculate the \(\chi^{2}\)-value
  2. How many degrees of freedom does this table have? Why?
  3. Using the \(\chi^2\) Table, what is the p-value?
  4. Are mode of transport and departmental association independent in the population?

The remaining questions can be answered with the help of the app below (it might take a moment to load, as it is a web-based application).

  1. Small effect, big N
    1. Build a table with a large total N but only a slight row difference (for example, a=210, b=190, c=190, d=210).
    2. Note the χ² p-value. Now scale the same proportions down (for example, a=21, b=19, c=19, d=21).
    3. What happens to the \(\chi^2\) p-value as N changes, and why? Use “Row %” to confirm the effect size stayed about the same.
  2. Same row-percentage gap, different base rates
    1. Construct two tables with a similar difference between the rows’ “Yes” percentages but very different column totals (base rates).
      • Example A (balanced): a=60, b=40, c=40, d=60 (N=200).
      • Example B (skewed): a=90, b=10, c=70, d=30 (N=200).
    2. Compare χ² p-values and the “Row %” and “Column %” views. How do base rates (margins) influence the test, even when the row-percentage gap is similar?
  3. Exploring Fisher’s exact test
    1. Build some very small tables (with low cell counts). Try cases where at least one expected count is less than \(5\), and cases where all expected counts are above \(5\). Watch for when the app switches from showing a \(\chi^2\) p-value to also showing a Fisher p-value.
    2. Compare the \(\chi^2\) and Fisher results in these tiny-\(N\) situations. Do they always agree? When do they differ most?
    3. Based on your experiments, summarize in your own words:
      • What condition seems to trigger Fisher’s test to appear?
      • Why might researchers prefer Fisher’s exact test in those situations?
      • In larger samples, why is \(\chi^2\) usually preferred instead?

Correlation

I have written yet another app! Once it loads (it will take a little time, it’s a beast), it lets you generate and explore bivariate relationships and see how Pearson’s r responds to different shapes, amounts of noise, and sample sizes. You can overlay a straight-line fit (the regression line we will explore next week) and a smooth non-linear fit (LOESS) to compare them. The current Pearson r is shown as a label on the plot.

Please note that in the explanations below I will refer to some relationships as “monotonic”, so here is a brief definition:

Monotonic means “always moving in one direction”.

A relationship is monotonic if, as x increases, y never reverses direction: it either never decreases (monotone increasing) or never increases (monotone decreasing). The rate can change and the curve can be curved or flat in places – it just must not switch from increasing to decreasing (or vice versa).

By the way, in the lecture we have focused on continuous data and this is preset in the app. But from the reading you (hopefully) know that other correlation coefficients exist for categorical data. You can explore an example for binary variables in the app, as well.

Getting started

  1. Under Data type, choose Continuous (scatter).
  2. Pick a Relationship pattern:
    • Linear (bivariate normal): straight-line trend with a controllable target r.
    • Quadratic: U-shaped, clearly non-linear and non-monotonic.
    • Cubic: monotonic but curved, a single overall direction with curvature.
  3. Adjust Sample size (n) and either Target Pearson r (for Linear) or Noise (for Quadratic and Cubic).
  4. Toggle the line of best fit (regression line) and LOESS smooth to compare linear versus non-linear fits.
  5. Use Seed to regenerate the sample while keeping settings the same.
  6. Read the Pearson r label on the plot.

Controls at a glance

  • Relationship pattern
    • Linear (bivariate normal): set Target Pearson r.
    • Quadratic and Cubic: set Noise to make the pattern tighter or looser.
  • n (Sample size): larger n stabilizes estimates.
  • Seed: changes the random draw while keeping other settings fixed.
  • Show least-squares line: straight-line fit.
  • Show smooth (LOESS): flexible curve that can reveal non-linear structure.

Tips for interpretation

  • Pearson’s r captures linear association.
  • Quadratic is not monotonic, so r can be near zero even when a strong U-shape is visible.
  • Cubic is monotonic but curved; r can be pulled down by curvature even when a strong relationship exists.
  • LOESS helps reveal curvature and local structure that a straight line can miss.
  • Increasing n reduces sampling variability; increasing Noise weakens visible structure.

Exercises

I. Linear relationships

  1. Set Linear, Target r = 0.60, n = 50. Record the Pearson r on the plot. Now set n = 300. How does r change, and what does this show about sample size?
  2. With Linear, try r = 0.20, n = 60. Change Seed several times. How much does r bounce around, and why?

II. Non-linear and non-monotonic (Quadratic)

  1. Choose Quadratic, set Noise = 0.10, n = 200. Is r near zero despite a clear U-shape? Explain why.
  2. Increase Noise until the U-shape is hard to see. At what point do the linear and LOESS fits become similarly uninformative?

III. Non-linear but monotonic (Cubic)

  1. Choose Cubic, set Noise = 0.15, n = 200. Compare the linear line to the LOESS curve. Does the straight line understate the visible relationship?
  2. Increase Noise gradually. How quickly does r decline as the pattern gets noisier? What does the LOESS curve reveal as noise grows?

IV. Comparing fits (lines)

  1. For Cubic with moderate Noise, toggle Least-squares line on and off and compare to LOESS. How does the straight line mislead about strength or direction?
  2. In Linear mode, when do the LOESS and linear fits essentially coincide, and why should they?

V. Sampling variability and reproducibility

  1. Fix a setting, for example Linear r = 0.40, n = 80, and change Seed a few times. How much does r vary? Repeat with n = 400 and compare.
  2. For Cubic with low Noise, change Seed repeatedly. How stable is r across seeds?

VI. Same r, different story

  1. Find two scenarios with similar Pearson r but different shapes (for example, Linear versus Cubic with noise tuned). How do the plots and the LOESS curve help you tell them apart?

VII. Reflections

  1. When is Pearson’s r an adequate summary, and when would you favor a non-linear description?
  2. Based on what you see in the app, how would you justify using a non-linear model rather than a linear one?

Takeaways

  • Pearson r summarizes linear, directional association.
  • Nonlinear, non-monotonic relationships (e.g., U-shapes) can be strong yet have r ≈ 0. Always plot your data.
